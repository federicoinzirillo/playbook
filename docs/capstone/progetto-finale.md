# Capstone — un sistema AI end-to-end

L'esame finale: un caso che attraversa **tutte** le macroaree. Non si risolve leggendo, si risolve costruendo e decidendo.

## Lo scenario
Un'azienda media vuole un assistente che risponda a domande sui propri documenti interni (contratti, policy, manuali), usabile dai dipendenti.

## Cosa devi produrre
1. **Scelta architetturale** — build vs buy, modello, perché _(sez. 5)_
2. **Sistema di retrieval** — chunking, vector store, hybrid/rerank, giustificati _(sez. 1)_
3. **Piano di valutazione** — golden dataset + criteri + soglie _(sez. 2)_
4. **Sicurezza** — prompt injection, accessi, dati sensibili _(sez. 3)_
5. **Produzione** — serving, costi stimati, monitoring _(sez. 4)_
6. **Decision record** — un documento che difende le scelte davanti a stakeholder

## Come valutarti
Se sai difendere ogni scelta nominando l'alternativa scartata e *perché*, sei pronto. Questo è il pezzo da mostrare a un colloquio.
