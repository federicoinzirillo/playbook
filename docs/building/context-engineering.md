# Context engineering: cosa mettere nel contesto, a scala

<p class="lesson-lead">Competenza distinta dal RAG classico. Con context window enormi, *decidere cosa entra nel contesto* è un mestiere a sé: il trade-off tra riempire il contesto e il degrado di attenzione nel mezzo (*lost in the middle*), più il costo che cresce col contesto. Aggancio al triangolo qualità-latenza-costo (5.3).</p>
